2. Getting started

You will need an API key to authenticate the requests. Luckly for you the examples below have been populated with a working key, so you can just copy and paste to see how it works.

That said, we strongly recommend you create a Free account to get your own unique API key. This allows you to use the service at will and to its full extend.

To send an order for analysis you will need to paste our JavaScript snippet and call our order API.

Sandbox predictable responses

Our Sandbox environment allows you to test your integration with our API and get predictable responses based on the decimals of total_amount.

total_amount response
*.00 Approve.
Score between 0 and 0.3.
*.01 to *.50 Review.
Score between 0.31 and 0.6.
*.51 to *.99 Decline.
Score between 0.61 and 0.99.
 

JavaScript snippet

Our Buying Behavior service works at it's best if you combine our JavaScript snippet with the API requests.

The snippet is used to collect browsing information from your visitors so we can draw a pattern of behavior for them.

When the visitor decides to make a purchase you need to send us their visitor_id so we can match their order with their browsing path.

Yours and your visitor's privacy are incredibly important to us. We will never share any information with third parties unless you, the merchant, explicitly asks us to. For more information please view our privacy policy.

Snippet example

<script type="text/javascript">
  var __kdt = __kdt || [];
  // Set your public key so we know it's you
  __kdt.push({"public_key": "TEE973BEC88"});
  // Optionally set the customer_id as well
  __kdt.push({"customer_id": "278393793"});
  (function() {
  var kdt = document.createElement('script');
  kdt.id = 'kdtjs'; kdt.type = 'text/javascript';
  kdt.async = true; 
  kdt.src = 'https://i.konduto.com/k.js';
  var s = document.getElementsByTagName('body')[0];
  s.parentNode.insertBefore(kdt, s);
  })();
</script>

Recover the visitor ID

<script type="text/javascript">
  var visitor_id = getKondutoID();
</script>

3. Authentication

In order to send requests you need to identify yourself using your API key, which can be obtained in your dashboard.

We use HTTP Basic Auth to authenticate merchants. The API key needs to be passed as your basic auth username and we don't require a password.

Please note that all requests must be made over HTTPS. If requests are received over plain HTTP we will reply with a 400 Bad Request error.

API Endpoint and Resources

POST https://api.konduto.com/v1/orders
PUT  https://api.konduto.com/v1/orders/{order_id}
GET  https://api.konduto.com/v1/orders/{order_id}

Setting your API key

$ curl -u T738D516F09CAB3A2C1EE: -X POST \
 -H 'Content-Type: application/json; charset=UTF-8' \
 -d '{}' https://api.konduto.com/v1/orders
 

4. Status codes & errors

Our API uses stardard HTTP response codes to indicate the result of a request. Successful requests will be replied with a 200 code while failed requests will contain other response codes.


Parameter Description
status Request status message.
Responds ok for successul and error for failed requests.
where Resource where the error occurred.
If / then the error is in the root object.
why Object containing the reason for the failed request.
why.expected (optional)
Expected value to be received in this request.
This can be a string or an array of values.
why.found (optional)
Actual value we received in this request.
why.missing (optional)
Required parameter missing from the request.

HTTP Response Codes

200 OK - Request was valid and we have a response.

400 Bad Request - Problem with the parameters sent. Please see our response for more information.

401 Unauthorized - No valid API key was received.

403 Forbidden - Problem with your account. Please contact our Support team for more information.

404 Not Found - Resource requested was not found.

405 Method Not Allowed - Method sent is not allowed for this resource.

500, 502, 503 Server Error - Something went wrong on our end. Please contact our Support team for more information.

Error response example

HTTP Status Code: 400 Bad Request

{
  "status": "error",
  "where": "/total_amount",
  "why": {
    "expected": ["integer","number"],
    "found": "string"
  }
}

The Payment Array

This array of objects should be used to send information about the payment method used to make the purchase.

You can send a transaction that contains multiple payment methods, such as purchases with two different credit cards.

Currently we only support credit card transactions, so you should check with support about using Konduto with other payment types.

Our API does not accept plaintext credit card numbers. If you are not using one of our SDKs you must hash the customer's card number with SHA1 before submitting the order.

There's no need to hash the values if you are using our SDK, as it already does that. Please note that the bin and last4 fields must not be hashed.

Parameter Description
type Payment type used by the customer. Currently only credit is supported.
Max. 6 chars, string.
sha1 Hash (SHA1) of the customer's full credit card number.
Exactly 40 chars, string.
bin First six digits of the customer's credit card. Used to identify the type of card being sent.
Exactly 6 chars, string.
last4 Four last digits of the customer's credit card number.
Exactly 4 chars, string.
expiration_date (optional)
Card's expiration date under MMYYYY format.
Exactly 6 chars, string.
status (optional)
The status of the transaction returned by the payment processor.
Can be approved, declined or pending if the payment wasn't been processed yet.
Max. 8 chars, string.

6. Update an order

You can update the status of a transaction programatically through the PUT method and passing a JSON in the request's body.

The request consists of simple object containing two parameters, status and comments.

You can also update the status of a transaction in our dashboard.

Parameter Description
status
 New status for this transaction.
Either approved, declined or fraud, when you have identified a fraud or chargeback.
Max. 8 chars, string.
comments
 Reason or comments about the status update.
Max. 255 chars, string.

Request example

$ curl -u T738D516F09CAB3A2C1EE: -X PUT \
 -H 'Content-Type: application/json; charset=UTF-8' \
 -d '{"status": "approve",
      "comments": "Talked to the customer and confirmed 
      their billing address."
      }' \
https://api.konduto.com/v1/orders/{order_id}

Response example

HTTP Status Code: 200 OK

{
  "old_status": "review",
  "new_satus": "approve"
}

What are categories?

Product categories are important because they tell us what type of items the customer is buying (shoes, jewelry or a GPS). Getting a TV is very different from getting a book, so categories matter for us.

These are loosely based on Google's product taxonomy and they represent the main categories in which you can group items into.

If you think there's a category missing feel free to contact us at hi@konduto.com and let us know!
