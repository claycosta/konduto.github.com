2. Primeiros passos

Você vai precisar de uma chave de API para autenticar as requisições. Para facilitar, os exemplos abaixo já estão populados com uma chave que funciona, assim você só precisa copiar e colar para ver como funciona.

Apesar disso, nós recomendamos que você crie uma conta Grátis para ter a sua própria chave única. Isso permite que você use o serviço à vontade e em toda sua capacidade.

Para enviar um pedido para análise você precisa colar nosso código JavaScript e chamar a nossa API de pedidos.

Respostas do ambiente de teste

Nosso ambiente de teste (Sandbox) permite que você teste a integração com a nossa API e receba respostas pré-definidas, que estão baseadas nos decimais do campo total_amount.

total_amount resposta
*.00 Aprovar.
Score entre 0 e 0.3.
*.01 to *.50 Revisar.
Score entre 0.31 e 0.6.
*.51 to *.99 Negar.
Score entre 0.61 e 0.99.
 

Código JavaScript

Nosso serviço de Buying Behavior funciona melhor quando você combina o nosso JavaScript com as chamadas à API.

O JavaScript é usado para coletar informações de navegação dos seus visitantes para que possamos determinar um padrão de comportamento para eles.

Quando o visitante decide fazer uma compra você precisa nos enviar o visitor_id deles, assim podemos cruzar o pedido com a navegação.

Exemplo do JavaScript

<script type="text/javascript">
  var __kdt = __kdt || [];
  // Defina sua chave pública para sabermos que é você
  __kdt.push({"public_key": "TEE973BEC88"});
  // Defina também o customer_id do cliente (opcional)
  __kdt.push({"customer_id": "278393793"});
  (function() {
  var kdt = document.createElement('script');
  kdt.id = 'kdtjs'; kdt.type = 'text/javascript';
  kdt.async = true; 
  kdt.src = 'https://i.konduto.com/k.js';
  var s = document.getElementsByTagName('body')[0];
  s.parentNode.insertBefore(kdt, s);
  })();
</script>

Recupere o ID do visitante

<script type="text/javascript">
  var visitor_id = getKondutoID();
</script>

3. Autenticação

Para poder enviar pedidos você precisa se identificar usando a nossa chave de API, que pode ser obtida no seu painel.

Nós usamos HTTP Basic Auth para autenticar os lojistas. A chave de API precisa ser passada como seu nome de usuário (username), e nós não pedimos senha.

Por favor note que todas as chamadas devem ser feitas via HTTPS. Se as chamadas forem feitas via HTTP normal nós responderemos com um erro 400 Bad Request.

Endpoint de API e Recursos

POST https://api.konduto.com/v1/orders
PUT  https://api.konduto.com/v1/orders/{order_id}
GET  https://api.konduto.com/v1/orders/{order_id}

Definindo sua chave de API

$ curl -u T738D516F09CAB3A2C1EE: -X POST \
 -H 'Content-Type: application/json; charset=UTF-8' \
 -d '{}' https://api.konduto.com/v1/orders
 

4. Códigos de status e erros

Nossa API usa códigos de resposta HTTP para indicar o resultado de uma chamada. Uma requisição bem sucedida retornará o status 200 enquanto que requisições que falharam terão outros códigos de resposta.


Parêmetro Descrição
status Mensagem de status da chamada.
Traz ok quando bem sucedida e error quando falhou.
where Recurso onde o erro ocorreu.
Se for / então o erro está no objeto raíz.
why Objeto que contém a razão da falha da chamada.
why.expected (opcional)
Valor que espera-se receber nesta chamada.
Isto pode ser uma string ou um array de valores.
why.found (opcional)
Valor que foi recebido nesta chamada.
why.missing (opcional)
Parâmetro obrigatório que está faltando na chamada.

Códigos de Resposta HTTP

200 OK - Chamada é válida e temos uma resposta.

400 Bad Request - Houve um problema com a requisição enviada. Por favor veja o corpo da resposta para mais informações.

401 Unauthorized - Não foi encontrada uma chave de API.

403 Forbidden - Há algum problema com a sua conta. Por favor entre em contato com o nosso Suporte para mais informações.

404 Not Found - O recurso solicitado não foi encontrado.

405 Method Not Allowed - O método HTTP enviado não é permitido para este recurso.

500, 502, 503 Server Error - Houve algum erro no nosso lado. Por favor entre em contato com o Suporte para mais informações.

Exemplo de resposta de erro

HTTP Status Code: 400 Bad Request

{
  "status": "error",
  "where": "/total_amount",
  "why": {
    "expected": ["integer","number"],
    "found": "string"
  }
}

O Array 'Payment'

Este array de objetos deve ser usado para enviar informações sobre os meios de pagamentos usados na compra.

Você pode mandar um pedido que contém múltiplos meios de pagamento, como compras com dois cartões de crédito.

Atualmente nós suportamos apenas compras com cartão de crédito. Consulte com o support a respeito de outros meios de pagamento para análise.

Nossa API não aceita o envio de números de cartões abertos. Se você não estiver usando nossos SDKs você deve fazer um hash do número de cartão de crédito do cliente usando SHA1 antes de submeter a compra para nós.

Não é necessário fazer hash se você estiver usando um dos nossos SDKs, pois eles já fazem isso. Por favor note que os campos bin e last4 não devem ser hasheados.

Parâmetros Descrição
type Meio de pagamento usado pelo cliente. Atualmente suportamos apenas credit
Max. 6 chars, string.
sha1 Hash SHA1 do cartão de crédito completo do cliente.
Exatamente 40 chars, string.
bin Os primeiros seis números do cartão de crédito. É usado para identificar o tipo de cartão usado.
Exatamente 6 chars, string.
last4 Os últimos quatro dígitos do cartão de crédito.
Exatamente 4 chars, string.
expiration_date (opcional)
Data de validade do cartão no formato MMAAAA
Exatamente 6 chars, string.
status (opcional)
O status do pagamento retornado pela operadora de cartão.
Pode ser approved (aprovado), declined (negado) ou pending (pendente), caso o pagamento não tenha sido feito ainda.
Max. 8 chars, string.

6. Atualizar um pedido

Você pode atualiar o status de um pedido programáticamente através do método PUT e passando um JSON no corpo da requisição.

A chamada consiste em um objeto simples que contém dois campos, status e comments.

Você também pode atualizar o status de um pedido no nosso dashboard.

Parâmetros Descrição
status
 Novo status deste pedido.
Pode ser approved (aprovado), declined (negado) ou fraud (fraude), quando você identificou uma fraude ou chargeback.
Max. 8 chars, string.
comments
 Razão ou comentários sobre a atualização do status.
Max. 255 chars, string.

Exemplo de chamada

$ curl -u T738D516F09CAB3A2C1EE: -X PUT \
 -H 'Content-Type: application/json; charset=UTF-8' \
 -d '{"status": "approve",
      "comments": "Talked to the customer and confirmed 
      their billing address."
      }' \
https://api.konduto.com/v1/orders/{order_id}

Exemplo de resposta

HTTP Status Code: 200 OK

{
  "old_status": "review",
  "new_satus": "approve"
}

O que são as categorias?

Categorias de produto são importantes pois elas nos dizem o tipo de produto que o cliente está comprando (sapatos, jóiais ou um GPS). Comprar um TV é muito diferente de comprar um livro, então as categorias fazem diferença para nós.

Esta lista foi baseada na taxonomia de produtos do Google e elas representam os principais agrupamentos de produtos.

Se você acha que há alguma categoria faltando fique à vontade para nos contactar no hi@konduto.com e nos avisar!
